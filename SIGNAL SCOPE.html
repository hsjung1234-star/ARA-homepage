Part 01 — Problem

동기화가 깨질 때 생기는 문제들

배선인지, 노이즈인지, 설정인지 — 신호를 보기 전엔 알 수 없습니다.

A-01
과검 속출

조명과 카메라 박자가 1ms만 어긋나도 정상 제품이 불량으로 판정됩니다. 이미지가 어둡거나 흐릿하게 촬영됩니다.

A-02
고속 트리거 유실

장비가 빨라지면 OS 기반 시스템은 신호 속도를 따라가지 못해 특정 구간의 촬영을 아예 놓칩니다.

A-03
유령 트리거(Ghost)

미세한 진동·노이즈를 실제 신호로 오인하여 제품이 없는데도 촬영. Backlash 오차도 같은 원인입니다.

→ 검사 신뢰성 하락 정상 제품 불량 판정 · 촬영 누락 · 유령 트리거 반복
B-01
OS 스케줄링 지연

Windows OS가 백그라운드 작업을 처리하느라 트리거를 제때 내보내지 못합니다. ±5~50ms 타이밍 오차.

B-02
노이즈 간섭 취약

디지털 노이즈 필터 없이는 주변 장비의 전기적 노이즈를 실제 신호로 오인합니다. GND 미연결 시 더 심각.

B-03
장비 보호 실패

전기적 절연 없이 외부 단락 발생 시 고가의 카메라·Frame Grabber까지 함께 손상될 수 있습니다.

→ 시스템 성능 저하 나노초 타이밍 불가 · 노이즈 오인 · 장비 손상 리스크
C-01
원인 불명 누락

촬영이 안 될 때 배선인지, 노이즈인지, 설정인지 파악할 길이 없습니다. 신호를 눈으로 볼 수 없기 때문입니다.

C-02
트러블슈팅 장기화

신호를 눈으로 볼 수 없어 문제 해결에 일주일 이상 소비됩니다. 라인 가동 중단 손실이 누적됩니다.

C-03
AI 학습 데이터 오염

조명 전압 강하로 어둡게 찍힌 이미지를 불량으로 학습하면 AI 모델 정확도가 급감합니다.

→ 유지보수 장기화 원인 특정 불가 · 일주일+ 라인 정지 · AI 모델 정확도 급감
